  0: Machine Learning for Natural Language
  1: Processing: A Comprehensive Review
  2: Abstract -- This position paper compares three transformer architectures -- BERT, Longformer, and BigBird -- across
  3: standard-length text classification and long-document understanding tasks. BERT employs full self-attention with quadratic
  4: complexity, limiting its practical sequence length. Longformer introduces a combination of local windowed attention and global
  5: attention tokens to achieve linear scaling. BigBird combines random attention, local attention, and global tokens to approximate
  6: full attention while maintaining theoretical guarantees of universal approximation. The architectural trade-offs center on attention
  7: sparsity patterns, computational complexity, and the ability to model long-range dependencies without prohibitive memory
  8: requirements.
  9: Index Terms -- machine learning, natural language processing, deep learning, transformers, BERT
 10: Introduction
 11: Transformer architectures have become the dominant
 12: paradigm in natural language processing, with BERT
 13: establishing a foundational bidirectional encoder
 14: architecture that demonstrated particular strength in
 15: understanding contextual relationships across
 16: standard-length sequences. However, the quadratic
 17: computational complexity of full self-attention creates a
 18: fundamental bottleneck for processing long documents.
 19: This limitation has motivated sparse attention variants
 20: including Longformer and BigBird, which modify the
 21: attention pattern to achieve linear or near-linear scaling
 22: with sequence length. The present analysis examines how
 23: these architectural choices affect performance on two
 24: distinct task regimes: standard-length text classification,
 25: where full attention remains tractable, and long-document
 26: understanding, where sequence length exceeds the
 27: practical limits of dense attention.
 28: Architectural Comparison
 29: BERT utilizes full self-attention where every token
 30: attends to all other tokens in the sequence, resulting in
 31: O(n?) time and memory complexity with respect to
 32: sequence length n. This design captures complete
 33: pairwise token interactions but constrains maximum
 34: sequence length to typically 512 tokens in practice.
 35: Longformer replaces the dense attention matrix with a
 36: sparse pattern combining local windowed attention --
 37: where each token attends to a fixed-size neighborhood --
 38: with global attention tokens that attend to and are
 39: attended by all tokens. This yields O(n) complexity while
 40: preserving the ability to aggregate information across the
 41: full sequence through designated global tokens. BigBird
 42: employs a three-component attention mechanism: local
 43: attention similar to Longformer, random attention where
 44: each token attends to a small set of randomly selected
 45: tokens, and global tokens analogous to Longformer's
 46: global attention. The random attention component
 47: provides theoretical guarantees that BigBird can
 48: approximate any continuous sequence-to-sequence
 49: function and is a universal approximator, while
 50: maintaining O(n) complexity.Task 1: Standard-Length Text Classification
 51: For standard-length text classification tasks where
 52: sequence lengths remain within BERT's 512-token limit,
 53: all three architectures can process inputs without
 54: truncation. BERT's full attention provides complete
 55: token-to-token interaction, which may capture
 56: fine-grained contextual relationships beneficial for
 57: classification tasks requiring holistic sentence
 58: understanding. Longformer's local windowed attention
 59: restricts direct token interactions to a fixed window,
 60: potentially limiting the model's ability to integrate distant
 61: but relevant context within a single layer, though global
 62: tokens can mitigate this by serving as aggregation points.
 63: BigBird's random attention introduces stochastic
 64: connectivity that may help propagate information across
 65: the sequence more effectively than purely local attention,
 66: while global tokens provide dedicated pathways for
 67: sequence-level representation. The computational
 68: trade-off favors BERT in this regime, as the quadratic
 69: cost remains manageable and the dense attention pattern
 70: avoids the inductive biases introduced by sparsity.
 71: Task 2: Long-Document Understanding
 72: Long-document understanding tasks involve sequences
 73: that exceed the practical limits of dense attention, making
 74: sparse attention architectures necessary. Longformer's
 75: local-plus-global pattern enables processing of sequences
 76: of thousands of tokens by constraining most attention to
 77: local windows while using global tokens -- typically
 78: placed at the beginning of the sequence or at segment
 79: boundaries -- to capture document-level information.
 80: This design is effective when document structure aligns
 81: with the global token placement, such as classification
 82: tasks where a [CLS]-like global token aggregates
 83: information. BigBird's addition of random attention
 84: provides alternative pathways for information flow that
 85: do not depend on predefined global token positions,
 86: potentially offering more robust long-range dependency
 87: modeling for tasks like long-form question answering
 88: where relevant information may appear at arbitrary
 89: positions. Both architectures achieve linear scaling, but
 90: their inductive biases differ: Longformer assumes
 91: important long-range interactions are mediated through
 92: designated global positions, while BigBird assumes
 93: random connectivity suffices to approximate full
 94: attention. The choice between them depends on whether
 95: the task benefits from structured global aggregation or
 96: distributed long-range connectivity.

  0: Discussion
  1: The architectural comparison reveals a fundamental
  2: trade-off between attention expressivity and
  3: computational scalability. BERT's dense attention
  4: maximizes representational capacity for short sequences
  5: but cannot scale to long documents. Longformer and
  6: BigBird both achieve linear complexity through sparsity,
  7: but encode different assumptions about how long-range
  8: dependencies should be modeled. Longformer's global
  9: tokens create an explicit hierarchical information flow
 10: suitable for tasks with clear document-level aggregation
 11: points. BigBird's random attention provides a more
 12: uniform approximation of full attention with theoretical
 13: universality guarantees, potentially better suited for tasks
 14: requiring flexible long-range reasoning. For
 15: standard-length classification, the sparsity-induced
 16: inductive biases of Longformer and BigBird may
 17: constrain performance relative to BERT's unrestricted
 18: attention, though the gap narrows with sufficient model
 19: capacity and training data. For long-document tasks, the
 20: choice between Longformer and BigBird should be
 21: guided by task structure: classification-style tasks with
 22: natural aggregation points align with Longformer's global
 23: token design, while open-ended reasoning over long
 24: contexts may benefit from BigBird's distributed
 25: connectivity.
 26: Conclusion
 27: BERT, Longformer, and BigBird represent three points
 28: on the attention sparsity spectrum, each optimized for
 29: different sequence length regimes and task structures.
 30: BERT remains the natural choice for standard-length
 31: tasks where quadratic attention is tractable and full token
 32: interaction is beneficial. Longformer's local-plus-global
 33: pattern offers an efficient and interpretable solution for
 34: long documents with hierarchical structure. BigBird's
 35: random-local-global combination provides a theoretically
 36: grounded approximation of full attention with flexible
 37: long-range modeling. The architectural decision should
 38: be driven by sequence length constraints, task structure,
 39: and the nature of long-range dependencies required by
 40: the target application.
